### Sprint Goal
Développer la structure hiérarchique et le responsive du site vitrine (carte de visite) en reprenant le contenu d'une version existante développée sous Bubble.

#### US-3.1 : Création d'un JSON exemple (mockup) pour concevoir le responsive
- **En tant que** Développeur
- **Je souhaite** Créer un fichier JSON exemple contenant une structure de données mockup représentant la structure hiérarchique du site vitrine
- **Afin de** Pouvoir concevoir et développer le responsive des composants sans dépendre du contenu réel
- **Critères d'acceptation** :
- Le fichier JSON est créé dans un dossier approprié (ex: `data/`)
- La structure JSON reflète la hiérarchie du site vitrine (sections, sous-sections, contenus)
- Le JSON contient des données d'exemple réalistes mais factices (texte lorem ipsum, images placeholder, etc.)
- La structure permet de représenter tous les types de contenus du site (texte, images, listes, etc.)
- Le format JSON est valide et bien structuré
