### Sprint Goal
Développer la structure hiérarchique et le responsive du site vitrine (carte de visite) en reprenant le contenu d'une version existante développée sous Bubble.

#### US-3.1 : Affichage d'un contenu mockup dans le navigateur pour validation de la mise en page
- **En tant que** Product Manager
- **Je souhaite** Voir s'afficher dans le navigateur un contenu mockup (données d'exemple) représentant la structure hiérarchique du site vitrine
- **Afin de** Valider la mise en page et la structure avant d'injecter le contenu réel
- **Critères d'acceptation** :
- Le contenu mockup s'affiche dans une page du navigateur
- La structure affichée reflète la hiérarchie du site vitrine (sections, sous-sections, contenus)
- Le contenu mockup contient des données d'exemple réalistes mais factices (texte lorem ipsum, images placeholder, etc.)
- Tous les types de contenus du site sont représentés (texte, images, listes, etc.)
- Le contenu est chargé depuis un fichier JSON (backend pur) et affiché via un composant React
