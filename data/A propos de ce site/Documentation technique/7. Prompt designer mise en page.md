# Prompt — Designer : proposition de mise en page du site

Ce document sert à invoker l’agent **Designer** (Task, `subagent_type: "Designer"`) pour qu’il propose et applique une mise en page cohérente sur l’ensemble du site, à partir du DOM RAW existant.

---

## Contexte

Le site est désormais rendu en **mode RAW** : structure HTML sémantique avec classes racine et `data-layout`, **sans feuille de style** appliquée aux types de contenu. La page de référence est **`/raw`** (en dev : `http://localhost:3000/raw`). Ton brief et tes ressources sont dans **`.cursor/agents/5. Designer.md`** (guide DOM, tokens, règle CSS, hiérarchie, effets visuels et interactifs à prendre en charge).

---

## Mission

**Proposer une mise en page (layout + présentation) cohérente pour tout le site**, en t’appuyant uniquement sur le DOM existant (classes racine, `data-layout`) et en respectant ton brief.

### Livrables attendus

1. **Proposition écrite courte** (à placer en tête de ta réponse ou dans un fichier dédié si tu le juges utile) :
   - Choix de **typographie** (utilisation des tokens existants : `--h1-font-size`, `--h2-color`, `--p-font-size`, etc. ; polices déjà chargées : Outfit, Clint Marker).
   - Choix de **couleurs et contrastes** (tokens `--CouleurFoncé`, `--CouleurClaire`, `--Noir`, `--Blanc`, etc.).
   - **Grilles et espacements** : colonnes, marges, largeur max texte (`--largeurDeTexteMax`), alignement des blocs (hero, profils, témoignages, domaines de compétences, détournements vidéo).
   - **Breakpoints responsive** : comportement mobile / tablette / desktop pour les listes (profils, témoignages, compétences, etc.) et pour le hero (gauche/droite, vidéo).
   - **Effets visuels et interactifs** documentés dans `app/raw/README.md` et `app/raw/README.md` : tooltip droits d’auteur, accordéon expériences, alternance de fond (`.light`), fond vidéo, taille des boutons (`.tailleGrande`).

2. **Application dans les fichiers CSS autorisés** :
   - **`app/content-styles.css`** : styles pour tous les types de contenu (hero, titre, titreDePage, texteLarge, video, domaineDeCompetence, competence, experienceEtApprentissage, callToAction, groupeDeBoutons, listeDesPages, blocsProfils, profil, temoignages, temoignage, listeDeDetournementsVideo, detournementVideo) en ciblant les **classes racine** et **data-layout** du guide `app/raw/README.md`.
   - **`app/globals.css`** : ajustement ou ajout de **tokens** uniquement si nécessaire (couleurs, tailles, typo) ; privilégier l’usage des variables existantes.

Tu **ne modifies pas** les composants React ni les utils. Si la structure DOM ou les données ne permettent pas le design souhaité, tu **demandes** explicitement les modifications au back-end ou au front-end.

---

## Rappels du brief

- **Un seul fichier** pour les types de contenu : `app/content-styles.css` (pas de `.module.css` pour hero, profil, etc.).
- **Cibler** les classes racine et `data-layout` documentés dans `app/raw/README.md` et la hiérarchie dans `data/A propos de ce site/Documentation technique/2. Hierarchie du site.md`.
- **Responsive** et **accessibilité** de base (contraste, lisibilité).
- C’est terminé quand : styles conformes au brief, rendu responsive et accessible, aucune régression sur la structure DOM (seuls les fichiers CSS sont modifiés).

---

## Texte à copier dans Task (prompt)

```
Le site est en mode RAW (DOM sans style). En t’appuyant sur ton brief (`.cursor/agents/6. Designer.md`), la page `/raw`, `app/raw/README.md`, `data/A propos de ce site/Documentation technique/2. Hierarchie du site.md`, `app/content-styles.css`, `app/globals.css` et `app/raw/README.md` :

1. Rédige une proposition de mise en page courte : typo (tokens + polices Outfit / Clint Marker), couleurs/contrastes, grilles et espacements, breakpoints responsive, et prise en charge des effets visuels/interactifs (tooltip droits d’auteur, accordéon expériences, .light, fond vidéo, .tailleGrande).

2. Applique cette proposition dans `app/content-styles.css` (et dans `app/globals.css` pour les tokens si besoin). Cible uniquement les classes racine et data-layout du guide RAW. Respecte la règle : un seul fichier global pour les types de contenu, pas de module.css. Responsive et accessibilité de base.

Si la structure DOM ne permet pas le design souhaité, indique clairement ce que l’équipe doit modifier (sans modifier toi-même les composants ou les utils).
```
