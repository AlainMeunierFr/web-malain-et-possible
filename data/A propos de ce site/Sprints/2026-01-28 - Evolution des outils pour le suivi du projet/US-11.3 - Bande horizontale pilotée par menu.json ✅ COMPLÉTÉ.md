# US-11.3 : Bande horizontale pilotée par menu.json (A propos de ce site)

## En tant que visiteur ou membre du projet qui consulte la page « A propos de ce site »

## Je souhaite que la bande horizontale soit pilotée par un fichier **menu.json** défini dans `data/A propos de ce site/menu.json`

## Afin de pouvoir ordonner les lignes de menu, mélanger dossiers (Path) et zones spécifiques (container), et inclure des liens vers les consignes agents (.cursor/agents)

# Critères d'acceptation

## CA1 - CA1 - Source des lignes de menu
- La bande horizontale n'affiche **plus** la liste des dossiers obtenue par scan du système de fichiers dans `data/A propos de ce site`.
- Elle affiche la **liste des lignes de menu** définies dans `data/A propos de ce site/menu.json`.

## CA2 - CA2 - Structure d'une ligne de menu (JSON)
- Chaque objet du JSON est une « ligne de menu » avec les propriétés :
- **Titre** : titre affiché dans la bande.
- **Numéro** : numéro d'ordre pour le tri des lignes (affichage selon ce tri).
- **Type** : type de contenu que cette ligne va afficher :
- **Path** : chemin vers un dossier ; comportement identique à l'actuel (page de visualisation de dossier, paramètre = chemin).
- **container** : zone spécifique (ex. « Sprint en cours ») ; le contenu est déterminé par le paramètre (ex. `sprintEnCours`).
- **Parametre** : paramètre nécessaire pour le type (chemin vers un dossier, nom du container, etc.). Les chemins sont relatifs à la racine du projet.

## CA3 - CA3 - Comportement selon le type
- **Type = Path** : au clic, ouverture de la page de visualisation de dossier (route dynamique) avec le dossier correspondant à **Parametre** ; affichage H2 = fichiers MD, H3+ = contenu.
- **Type = container** : au clic (ou sans navigation), affichage dans la zone sous la bande du container indiqué par **Parametre** (ex. `sprintEnCours`). Pas de changement d'URL vers un dossier.

## CA4 - CA4 - Exemple de contenu menu.json
- Le fichier `data/A propos de ce site/menu.json` contient au moins les lignes suivantes (ordre par Numéro) :
1. Titre = « A propos du projet », Numéro = 1, Type = Path, Parametre = `data/A propos de ce site/md/A propos du projet`
2. Titre = « Sprint en cours », Numéro = 2, Type = container, Parametre = `sprintEnCours`
3. Titre = « Definition Of Done (DOD) », Numéro = 3, Type = Path, Parametre = `data/A propos de ce site/md/Definition of Done (DOD)`
4. Titre = « Consignes aux agents », Numéro = 4, Type = Path, Parametre = `.cursor/agents`

**Contexte** : Affinage de l'US-11.2 : la bande n'est plus dérivée du scan des dossiers mais d'un fichier de configuration, permettant d'ajouter des entrées comme « Sprint en cours » (container) et « Consignes aux agents » (dossier .cursor/agents).
