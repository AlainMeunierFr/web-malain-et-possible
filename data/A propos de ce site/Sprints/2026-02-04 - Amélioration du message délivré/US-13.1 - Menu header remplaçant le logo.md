# US-13.1 : Menu header remplaçant le logo

## En tant que visiteur du site

## Je souhaite voir un menu de navigation dans le header à la place du logo, dont le contenu est configurable

## Afin d'accéder rapidement aux principales sections depuis n'importe quelle page, et que l'équipe puisse adapter les entrées de menu sans modifier le code

# Critères d'acceptation

## CA1 - Affichage menu desktop
- Le logo est retiré du header
- Un menu horizontal s'affiche à gauche du header avec des liens et des menus déroulants (dropdown)
- La photo reste en place à droite du header
- Le titre de page apparaît sous le header, sur fond blanc, en bleu, taille réduite pour préserver l'espace

## CA2 - Menu paramétrable
- Le contenu du menu (entrées principales et sous-menus) est piloté par la configuration, sans modification du code
- L'équipe peut définir l'ordre des entrées de menu et rattacher des pages à des sous-menus
- Par défaut, avec la configuration livrée :
  - Entrées principales : Accueil, Mes profils, Détournements, À propos de ce site
  - Comportement des entrées avec sous-menu : le survol ouvre le dropdown ; le clic sur le libellé navigue vers la page mère. "Mes profils" → clic = /mes-profils, survol = sous-menu (Produit logiciel, Opérations, Transformation Agile, Technologie) ; "Détournements" → clic = /detournement-video, survol = sous-menu (Portfolio)
  - Autres liens directs : Accueil (/), À propos (/a-propos-du-site)

## CA3 - Menu hamburger mobile
- Sur mobile (en dessous d'un seuil de largeur), le menu horizontal est masqué
- Une icône hamburger s'affiche à la place
- Au clic sur l'icône, un panneau latéral ou overlay s'ouvre avec les mêmes entrées que le menu desktop (liens directs et sous-menus)
- La photo reste visible dans le header sur mobile

## CA4 - Logo dans le footer
- Un bouton affichant le logo "Malain et possible" est ajouté dans le footer
- Le bouton est visible mais sans action pour le moment (réservé à un usage futur)

## CA5 - Structure du fichier de configuration
- Le fichier `_Pages-Et-Lien.json` est renommé en `_Pages-Liens-Et-Menus.json` (ou compatibilité avec l’ancien nom conservée)
- Une section `menus` est ajoutée avec : `menus.header` (entrées du menu, ordre et sous-menus) et `menus.exclusHeader` (pages visibles dans le plan mais explicitement exclues du menu header, ex. Pour aller plus loin, Faisons connaissance)
- Si `menus` est absent, le comportement par défaut (CA2) s’applique

## CA6 - Évolution pour la responsable des paramétrages
- Une seule source de vérité : pages, liens et menu header dans le même fichier
- La section `menus` est documentée (README ou commentaires dans le fichier)

## CA7 - Intégrité des paramétrages
- Chaque URL référencée dans `menus.header` doit exister dans `pages`
- Les pages visibles dans le plan (dessiner Oui, zone ≠ Masqué) qui ne sont ni dans `menus.header` ni dans `menus.exclusHeader` sont signalées comme potentiellement oubliées
- Un script ou utilitaire de vérification est fourni (exécutable en local ou en CI)
