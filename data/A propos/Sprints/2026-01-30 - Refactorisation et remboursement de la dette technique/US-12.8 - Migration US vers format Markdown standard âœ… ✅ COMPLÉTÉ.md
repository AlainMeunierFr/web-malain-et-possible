# US-12.8 : Migration des fichiers US vers format Markdown standard

## En tant que
développeur du projet

## Je souhaite
disposer d'un script de migration qui convertit les fichiers User Story existants du format à puces avec gras (`- **En tant que** ...`) vers un format Markdown standard avec headers (`## En tant que`)

## Afin de
simplifier le parsing des fichiers Markdown et aligner le format sur les conventions MD standards

# Critères d'acceptation

## CA1 - Script de migration fonctionnel
- Le script lit tous les fichiers `.md` dans `data/A propos/Sprints/`
- Le script détecte les fichiers US (pattern `US-X.Y - *.md`)
- Le script convertit le format actuel vers le format cible pour chaque section (En tant que, Je souhaite, Afin de, Critères d'acceptation, CA1/CA2/...)
- Le script préserve le contenu textuel sans perte d'information
- Git est utilisé comme mécanisme de backup (pas de backup automatique par le script)

## CA2 - Format cible respecté
- Le titre reste en H1 : `# US X.Y - Titre`
- Les sections deviennent des H2 : `## En tant que`, `## Je souhaite`, `## Afin de`
- Le texte de chaque section est sur la ligne suivante, sans gras
- La section `# Critères d'acceptation` est un H1
- Chaque thème de CA devient un H2 : `## CA1 - Titre`
- Les critères sous chaque thème restent des puces simples : `- sous-élément`

## CA3 - Exécution sécurisée
- Le script peut être lancé en mode "dry-run" pour prévisualiser les modifications sans écrire
- Le script affiche un rapport de conversion (nombre de fichiers traités, erreurs éventuelles)
- Le script ne modifie pas les fichiers qui ne correspondent pas au pattern US

## CA4 - Validation post-migration
- Après migration, les fichiers sont valides syntaxiquement (pas de Markdown cassé)
- Un test peut être lancé pour vérifier que tous les fichiers ont été convertis

## CA5 - Mise à jour du DOD
- Le fichier `DOD - Équipe.md` est mis à jour pour refléter le nouveau format d'écriture des US
- Les exemples dans le DOD utilisent le nouveau format avec headers

## CA6 - Adaptation des parsers
- Les parsers existants (`aboutSiteReader.ts`, `sprintBoardReader.ts`, etc.) sont adaptés pour lire le nouveau format
- Les parsers restent rétrocompatibles avec l'ancien format pendant la période de transition (optionnel)
- Les tests unitaires des parsers sont mis à jour pour valider le nouveau format

## CA7 - Rendu visuel cohérent
- Lors de l'affichage web, les mots-clés "En tant que", "Je souhaite", "Afin de" apparaissent en gras
- Le texte qui suit ces mots-clés n'est pas en gras
- Ce rendu est géré par le CSS/renderer, pas par le Markdown source
