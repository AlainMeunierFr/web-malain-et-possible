### Migration convention -cont (Option A) + .main-content-cont

#### Contexte
Suite aux échanges sur la nomenclature (ID vs classe, boby.count → convention unifiée) : migration complète vers la convention spec [TypeDeContenu].cont → DOM .[TypeDeContenu]-cont.

#### Résultat technique

**1. main-content : ID → classe**
- `id="main-content"` remplacé par `class="main-content-cont"` + `id="main-content-cont"` (conservé pour skip link accessibilité)
- Header.tsx : `href="#main-content-cont"`
- Sélecteurs CSS : `#main-content` → `.main-content-cont`
- Spec/hierarchy : `main-content` → `main-content-cont`

**2. Containers migrés vers -cont**
- **Composants** : HeroSection (hero-cont, hero-gauche-cont, hero-droite-cont), ProfilContainer (profil-cont), Temoignages (listeDeTemoignages-cont, temoignage-cont), ListeDesPages (listeDesPages-cont), BlocsProfils (listeDeProfils-cont), DomaineDeCompetences (domaineDeCompetence-cont, competence-cont, domaineDeCompetence-header-cont, domaineDeCompetence-competences-cont), pages (main-cont)
- **CSS** : content-styles.css, a-propos.css — tous les sélecteurs alignés
- **Tests** : charte-graphique, plan-du-site-zones, HeroSection, DomaineDeCompetences

**.main-content-cont — règles dans content-styles.css**
- `display: flex`, `flex-direction: column`, `min-height: 100vh`
- `padding-top`: position bas du logo + 5px = `calc(max(11px, 1.2vh) + max(99px, 10.83vh) + 5px)`
- `padding-bottom`: `max(50px, 5.47vh)`
- Enfant direct : `margin-top: auto`, `margin-bottom: auto`, `min-height: 4px`
  - Contenu court → centré verticalement
  - Contenu long → collé en haut

**css-debug.css — strictement bordure + fond**
- Uniquement `border` et `background` avec `!important`
- Pas de min-height, pas de masquage enfants

**Hiérarchie "containers only"**
- Exclusion de `listeDesPages.page.cont` (template de répétition, pas container de layout)
- `CONTAINERS_ONLY_EXCLUDE` dans siteHierarchyGenerator

**Spec / DDD**
- Renommage boby.count → main-content dans canonicalSpec, hierarchy, ModeLectureRenderer
- Arbre ASCII affiche `.main-content-cont` pour correspondance DOM

#### Confirmation
`.main-content-cont` enveloppe toutes les pages via le layout racine (`app/layout.tsx`). Les règles s'appliquent à Home, mes-profils, profils, a-propos, etc.
